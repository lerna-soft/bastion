## 1. HostEditScreen funcional

### Problema
`HostEditScreen` tiene botones "Connect" y "SAVE CONFIGURATION" con
`onClick = { }` — no guardan ni conectan. Editar no carga datos existentes.

### Criterios de aceptación

| # | Criterio | Verificación |
|---|----------|-------------|
| AC1 | Crear servidor nuevo → persiste en Room + SecretsStore | DB inspeccionable |
| AC2 | Editar servidor existente → campos pre-poblados desde DB | HostEditScreen muestra valores guardados |
| AC3 | Botón "SAVE CONFIGURATION" → guarda y navega al vault | Host aparece en lista |
| AC4 | Botón "Connect" → guarda + abre terminal SSH directo | Sesión SSH activa |
| AC5 | Validación: hostname y username requeridos | Botón deshabilitado si vacío |
| AC6 | Puerto inválido (<1 o >65535) → muestra error | Tooltip/toast |

### Gherkin

```gherkin
Historia de usuario:
Como usuario de Bastion
Quiero crear y editar configuraciones de servidor que realmente se guarden
Para poder conectarme a mis servidores SSH sin perder la configuración

Escenario: Crear servidor nuevo y guardar
  Dado que estoy en la pantalla "Add Server"
  Cuando lleno "server-01" / "10.0.0.5" / "22" / "root" / password="secreto"
  Y presiono "SAVE CONFIGURATION"
  Entonces el servidor se guarda en Room con el password cifrado en SecretsStore
  Y navego al vault donde el servidor aparece en la lista

Escenario: Editar servidor existente
  Dado que hay un servidor "server-01" guardado
  Cuando edito ese servidor
  Entonces los campos aparecen pre-poblados con los valores guardados
  Y puedo modificarlos y guardar

Escenario: Conectar y guardar al mismo tiempo
  Dado que estoy en "Add Server" con datos válidos
  Cuando presiono "Connect"
  Entonces el servidor se guarda en DB
  Y se abre una terminal SSH conectada

Escenario: Validación de campos
  Dado que estoy en "Add Server"
  Cuando dejo "Hostname" vacío
  Entonces el botón "SAVE CONFIGURATION" está deshabilitado
  Y el campo muestra borde rojo con mensaje "Required"
```

## 3. Reemplazar datos quemados

### Problema
Múltiples textos hardcoded en lugar de valores dinámicos:

| Archivo | Texto quemado | Debería ser |
|---------|--------------|-------------|
| SSHKeysScreen.kt:242 | `"4 Active"` | `"${keys.count { it.isActive }} Active"` |
| SSHKeysScreen.kt:459-481 | Stats cards (98%, 2 Keys, 245 Events) | `"${keys.size} Total"`, etc. |
| SettingsScreen.kt:538 | `"2.4.0-stable"` | `BuildConfig.VERSION_NAME` |
| SettingsScreen.kt:543 | `"V8.2.1-Core"` | `BuildConfig.VERSION_NAME` |
| SettingsScreen.kt:559 | `"2024"` | `"2026"` (year actual) |
| AppLayout.kt:366 | `"Admin User"` | Placeholder editable o `"User"` genérico |
| AppLayout.kt:374 | `"System Superuser"` | `"Administrator"` genérico |

### Criterios

| # | Criterio |
|---|----------|
| AC9 | Badge "Active" dinámico según `keys.filter { it.isActive }.size` |
| AC10 | Stats cards reflejan datos reales de la lista de keys |
| AC11 | Versión en About usa `BuildConfig.VERSION_NAME` |
| AC12 | Copyright usa año actual dinámico |
| AC13 | Sidebar user info: "Admin User" → "User" genérico |

### Gherkin

```gherkin
Escenario: Badge de keys activas dinámico
  Dado que hay 3 SSH keys (1 activa, 2 inactivas)
  Entonces el badge muestra "1 Active"
  Cuando agrego una nueva key activa
  Entonces el badge muestra "2 Active"

Escenario: Stats cards reflejan datos reales
  Dado que hay 3 SSH keys
  Entonces la card "Total Keys" muestra "3 Keys"
  Y la card "Active Keys" muestra "1 Key"
```
