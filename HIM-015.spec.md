---

## Auditoría (2026-07-08)

| # | Elemento | Problema | Acción |
|---|----------|----------|--------|
| A1 | Settings → "Security" (2FA, Session Timeout) | Se guarda en DB pero nada lo aplica — falsa sensación de seguridad | Ocultar del sidebar de Settings |
| A2 | Settings → "Notifications" (Webhook, Email Alerts) | Se guarda pero nunca se envía nada | Ocultar del sidebar de Settings |
| A3 | Settings → "API Management" | Bastion no tiene servidor propio; las keys no autentican nada (remanente del mockup Stitch original, pensado como dashboard web) | Ocultar del sidebar de Settings |
| A4 | Header: ícono "Notifications" (campana) | Mal etiquetado — en realidad abre el repo de GitHub, no hay sistema de notificaciones | Reemplazar por ícono/label de GitHub correcto (SÍ funciona, solo estaba mal etiquetado) |
| A5 | Header: botón "Connect" | No-op silencioso si `terminalSessions` está vacío | Deshabilitar visualmente cuando no hay sesiones |
| A6 | Command Palette (⌘K) + Quick Actions | Ya inalcanzable (nada activa `showCommandPalette=true`) — código muerto. Además sus 3 acciones escriben texto literal en el shell remoto en vez de navegar | Sin acción (ya oculto); documentado para no reactivarlo tal cual en el futuro |

No se toca: Settings → "General" (serverName/timezone/language) — son decorativos pero no
engañosos (no prometen ninguna acción activa), y Settings → "Appearance" (sí funcional, ya
verificado en HIM-005/006).

---

## Criterios de aceptación

| # | Criterio |
|---|----------|
| AC1 | El sidebar de Settings ya no muestra "Notifications", "Security" ni "API Management" |
| AC2 | El ícono del header ya no dice "Notifications" — ahora es claramente "GitHub" |
| AC3 | El botón "Connect" del header aparece deshabilitado (opacidad reducida, sin click) cuando no hay terminales abiertas |
