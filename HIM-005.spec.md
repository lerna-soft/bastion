# HIM-005 — Correcciones UI Bastion v1.1.8

**Prioridad:** Alta
**Estimación:** 8 puntos
**Estado:** Borrador
**Dependencias:** Ninguna

## 1. Remover tabs Sessions/Clusters/History del AppHeader

**Problema:** El header muestra 3 textos ("Sessions", "Clusters", "History") que no son interactivos ni funcionales. No tienen click handler, no representan tabs reales ni hay planes de implementarlos.

**Solución:** Eliminar el Row con esos 3 textos del AppHeader. El header debe quedar solo con: search input + botón Connect + icono notificaciones + avatar.

**Archivos:** `AppLayout.kt` (líneas ~587-714, sección AppHeader)

---

## 2. User section en sidebar: sin datos falsos + avatar blur

**Problema:** El sidebar muestra "User" / "Superuser" hardcodeados. No hay sistema de login/registro implementado.

**Solución:**
- Reemplazar el Row del usuario con un Box que muestre solo el avatar (icono Person) con efecto blur
- Sin texto de nombre/rol
- Sin icono de escudo
- Tooltip/placeholder: "Sign in to manage profile"

**Archivos:** `AppLayout.kt` (líneas ~447-492)

---

## 3. Status bar: fix edge-to-edge con insets

**Problema:**
- `MainActivity.kt` llama `enableEdgeToEdge()` (modo edge-to-edge) pero `Theme.kt` sobrescribe el color de la status bar con un sólido, contradiciendo el edge-to-edge
- El contenido de Compose no usa `WindowInsets.systemBars` → se dibuja debajo de la barra de sistema (reloj, batería, etc.)
- No hay manejo de `navigationBars` ni `systemBars`

**Solución:**
- Remover `enableEdgeToEdge()` de MainActivity (inconsistente con el diseño actual)
- O alternativamente: mantener edge-to-edge pero aplicar `Modifier.windowInsetsPadding(WindowInsets.systemBars)` en el contenedor raíz de AppLayout
- Mantener `statusBarColor` en Theme.kt para que coincida con el fondo del tema
- Agregar padding de sistema en AppLayout para que el contenido no se superponga con la barra de estado

**Archivos:** `MainActivity.kt`, `AppLayout.kt`, `Theme.kt`

---

## 4. Font size: conectar AppSettings → TerminalBridge → JS

**Problema:** El slider de font size en Settings persiste el valor en SQLite (`AppSettings.fontSize`) pero nunca se aplica al terminal. El JS de hterm define `setFontSize(size)` pero nadie lo llama desde Kotlin.

**Solución:**
- Agregar `setFontSize(size: Int)` a `TerminalBridge`
- Pasar `fontSize` desde `AppSettings` a `TerminalTab` como parámetro
- Llamar `bridge.setFontSize(fontSize)` al iniciar el terminal y cuando cambie el setting
- En JS, `setFontSize(size)` debe actualizar `term.prefs_.set('font-size', size)`

**Archivos:** `TerminalBridge.kt`, `TerminalTab.kt`, `AppLayout.kt`, `SettingsScreen.kt`

---

## 5. Theme persistence: guardar/recuperar ColorMode de DB

**Problema:** `AppSettings.colorMode` se guarda como string en SQLite, pero `MainActivity` siempre inicia con `ColorMode.DARK` sin leer el valor guardado.

**Solución:**
- En `MainActivity`, después de cargar `AppSettings` de la DB, inicializar `colorMode` con el valor guardado
- Si no hay valor guardado, usar `ColorMode.DARK` como fallback
- Al cambiar el tema, persistir el nuevo valor en DB

**Archivos:** `MainActivity.kt`, `AppSettings.kt`

---

## 6. SystemStatsPanel: quitar mock + placeholder + datos reales

**Problema:** CPU 12.4%, RAM 4.2/16GB, Disk 412/512GB, Network 1.2MB/s son valores hardcodeados. No representan el servidor real.

**Solución (fase 1 — placeholder):**
- Reemplazar todos los valores hardcodeados con "--" o "N/A"
- Mostrar "Not connected" cuando no hay sesión activa
- Mantener la estructura visual del panel (barras de progreso, etiquetas)

**Solución (fase 2 — datos reales):**
- Cuando hay una conexión SSH activa, ejecutar comandos periódicamente:
  - `top -bn1 | head -3` → CPU load
  - `free -m` → RAM usage
  - `df -h /` → Disk usage
  - `/sbin/ifconfig eth0 | grep bytes` → Network I/O
- Parsear la salida en Kotlin mediante regex
- Actualizar el panel cada 5 segundos
- Mostrar los valores reales en las barras de progreso

**Archivos:** `TerminalTab.kt`

---

## 7. Update progress bar: estilizar con % y número

**Problema:** El diálogo de descarga de actualización usa un `LinearProgressIndicator` sin estilo y no muestra porcentaje ni tamaño.

**Solución:**
- Estilizar el `LinearProgressIndicator` con colores del tema (StitchPrimaryContainer)
- Agregar texto con "XX% — YY.MB / ZZ.MB"
- Agregar animación suave
- Usar `progres = downloadProgress / 100f` del estado `UpdateState.Downloading(progress)`

**Archivos:** `MainActivity.kt` (diálogo de update)

---

## 8. Predeterminado: Dark theme neutral (no verde neón)

**Problema:** El usuario no le gusta el verde neón `#00ff41` como color principal. Actualmente es el `StitchPrimaryContainer` y se usa extensamente.

**Solución:**
- Agregar nuevo tema "Neutral Dark" que use azul/gris como accent en vez de verde
- Mover el tema verde neón (Stitch) a una opción en Settings (ej. "Stitch Green")
- El tema por defecto será "Neutral Dark"
- Renombrar opciones en Appearance:
  - DARK → "Neutral Dark" (default)
  - LIGHT → "Light"
  - MONOKAI → "Monokai"
  - SYSTEM → "System"
  - Agregar "Stitch Green" (el actual)
- BONUS: Agregar tema "OLED Dark" (fondo negro puro #000000) para ahorro de batería

**Archivos:** `Color.kt`, `Theme.kt`, `SettingsScreen.kt`

---

## 9. Notificaciones: análisis de posibilidades

**Consulta:** Las notificaciones in-app no están implementadas. La única notificación real es la de actualización de la app (diálogo forzado).

**Casos de uso identificados (sin implementar):**
- Conexión SSH reconectada automáticamente
- Tiempo de inactividad de sesión (timeout)
- Comando de larga duración completado
- Error de conexión

**Decisión del usuario:** Mantener solo la notificación de actualización de app. No implementar otras notificaciones por ahora.
